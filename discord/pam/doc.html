Help and introduction

By default, the bot uses / as command prefix. This means that commands are invoked like so: /command arguments....

In below documentation, all commands will be listed with default command prefix. Arguments will be listed as <argument>, denoting required argument; <argument...> denoting required catch-all argument (i.e. one that can take more than one value). Optional arguments will be denoted using [argument] and [argument...] for single and catch-all optional arguments. When typing commands, you must omit the <> and [] parts, meaning that if a command is denoted as /command <argument 1> [argument 2], you use it as /command value1 value2 or /command value1. If you want to pass an argument that contains spaces, wrap it in "", for instance: /command "value for argument 1" will invoke /command with a single argument, value for argument 1. This is not required for catch-all arguments. For instance, if a command is declared as /echo <message...>, you can invoke it as /echo Hello, this is a message with spaces!.

/help [module] Displays help. Invoking this command without any arguments will list available modules. Specifying a module will show all commands declared in specified module, that the invoker can use.

Server management

/guild rename <name...> Renames current server. This command requires manage server permission.
/guild info Displays information about current server. This command can be used by anyone.

Member management

/members report <member> <reason...> Reports a member to moderators. This command requires moderation log to be configured on the server (see /guild config modlog), and can be used by anyone.
/members mute <member> [duration] Mutes a member for a specified time. If duration is not specified, mute is permanent, and can only be lifted manually. This command requires configured mute role (see /guild config muterole), as well as moderation log (see /guild configure modlog), as well as kick members permission.
/members unmute <member> Unmutes a member. This command requires configured mute role (see /guild config muterole), as well as moderation log (see /guild configure modlog), as well as kick members permission.
/members kick <member> Kicks a member. This command requires moderation log to be configured on the server (see /guild config modlog), as well as kick members permission.
/members softban <member> Softbans a member. Softban is a ban and instant unban. Functionally, it works like kicking, except, since a ban occurs, it also removes messages of the kicked member. The member is free to rejoin once softbanned. This command requires moderation log to be configured on the server (see /guild config modlog), as well as kick members permission.
/members ban <member> [duration] Bans a member for a specified time. If duration is not specified, ban is permanent, and can only be lifted manually. This command requires moderation log to be configured on the server (see /guild config modlog), as well as ban members permission.
/members unban <member id> Unbans a member by his user ID. This command requires moderation log to be configured on the server (see /guild config modlog), as well as ban members permission.
/members prune Prunes inactive members. Inactive members are ones that have not logged in within 7 days, and were not assigned a role. This command requires kick members permission.
/members information Displays information about a member. This command can be used by anyone.
/members reason <case id> <reason...> Sets a moderator action's reason, as well as responsible moderator. This command requires configured mute role (see /guild config muterole), as well as moderation log (see /guild configure modlog), as well as kick members and ban members permissions.

Role management

/roles create <name...> Creates a new role with specified name. This command requires manage roles permission.
/roles delete <role> Deletes specified role. This command requires manage roles permission.
/roles edit <role> <parameters...> Edits specified role. Arguments are passed as argument1=value1 argument2=value2. Possible arguments are name permissions colour hoist mentionable. This command requires manage roles permission.
/roles information <role> Displays information about specified role. This command requires manage roles permission.
/roles list Lists roles defined in specified server. This command requires manage roles permission.
/roles adduser <member> <roles...> Adds specified member to specified roles. This command requires manage roles permission.
/roles removeuser <member> <roles...> Removes specified member from specified roles. This command requires manage roles permission.
